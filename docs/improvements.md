# 项目阶段改进记录

本文件用于按**开发阶段**记录项目演进过程，方便后续撰写课题报告、阶段总结和答辩材料。

写法原则：

- 每个阶段都回答 4 个问题：处于什么阶段、目标是什么、改了什么、有什么用
- 重点描述“系统是如何一步步推进的”，而不是按功能方向堆砌迭代清单
- 同一个改动只放在最主要的阶段，避免重复展开

---

## 一、文档说明

这份记录面向两类用途：

- 课题报告：可直接提炼“阶段目标 - 技术改动 - 阶段成果”
- 答辩展示：可快速说明系统从演示原型如何演进为实验平台

因此，这里不采用“建模改动列表”“功能改动列表”的写法，而采用“阶段成果记录”的写法。

---

## 二、项目阶段总览

当前项目大致经历了 5 个阶段：

| 阶段 | 核心目标 | 结果 |
|------|----------|------|
| 阶段一 | 让基础鸟群与捕食行为可以运行 | 形成单画布可交互 Boids 演示 |
| 阶段二 | 让抗捕食机制更有建模解释力 | 建立更合理的感知、警报、捕食者行为模型 |
| 阶段三 | 让实验可复现、可记录、可导出 | 建立实验工作流和数据输出能力 |
| 阶段四 | 让用户更容易看懂规则、机制和过程 | 建立教学展示和可视化解释能力 |
| 阶段五 | 让正式实验不依赖 GUI，且以 A/B 为基本单位 | 建立 A/B 一体化实验与无图形实验入口 |

---

## 三、阶段一：基础仿真可运行

### 阶段定位

这一阶段的任务是从“空白项目”推进到“能跑起来的基础仿真原型”。

### 阶段目标

- 建立 Boids 的三条基础规则
- 建立捕食者与猎物的最基本交互
- 让系统具备可实时观察的单画布运行能力

### 主要改动

- 建立基础鸟群个体模型与捕食者模型
- 建立分离、对齐、聚合三条基础规则
- 建立基础捕食追击与捕获逻辑
- 建立 Rust + WebAssembly + JavaScript + Canvas 的整体技术链路
- 建立参数控制、实时渲染和基本统计输出

### 阶段作用

这一阶段的作用是把项目从“概念设想”推进到“可运行原型”。  
没有这一阶段，就没有后续机制实验、A/B 对比和报告展示的基础。

### 可在报告中如何表述

项目首先完成了鸟群基础行为与捕食过程的可运行实现，形成了一个可交互、可观察的单画布仿真原型，为后续机制研究和实验工具化奠定基础。

---

## 四、阶段二：机制建模增强

### 阶段定位

这一阶段的任务是从“能跑”推进到“能解释机制”，使系统更适合表现抗捕食理论。

### 阶段目标

- 让 Many-Eyes、Confusion、Selfish Herd 等机制有更合理的建模支撑
- 提升捕食者行为和鸟群感知模型的真实性
- 引入更能支持实验分析的关键指标

### 主要改动

- 将鸟群感知从近似全向半径升级为“距离 + 视野角度”的定向感知
- 将警报传播从单层状态传播升级为“直接发现 + 邻居传播”的双层传播
- 将捕食者追击升级为搜索、锁定、攻击三状态
- 增加转向率、锁定帧数、锁定稳定性和混淆强度等机制
- 强化边缘目标选择与局部暴露风险，使自私兽群更容易解释
- 增加首次预警延迟、警戒覆盖率、平均锁定时长、平均混淆强度等指标

### 阶段作用

这一阶段使项目不再只是一个一般性的 Boids 动画，而是能够更清楚地承载课程中的抗捕食机制解释。  
系统开始具备“为什么会这样”的分析基础，而不仅仅是“看起来会这样”。

### 可在报告中如何表述

为了提高模型的解释力，项目在这一阶段重点增强了感知、警报传播和捕食者状态机，使多眼效应、混淆效应和自私兽群等机制可以通过更明确的参数和指标表现出来。

---

## 五、阶段三：实验工作流建立

### 阶段定位

这一阶段的任务是从“参数试玩型原型”推进到“可复现实验工具”。

### 阶段目标

- 让实验具备固定 Seed 和复现实验能力
- 让结果能够被记录、回看和导出
- 让实验流程不再依赖人工记忆和手工整理

### 主要改动

- 增加固定 Seed、当前 Seed 显示和基于 Seed 的重置能力
- 建立单次实验、批量实验、停止实验等实验控制流程
- 建立实验结果记录、最近实验列表和 JSON/CSV 导出
- 将仿真内核按 `flocking / predation / metrics / experiment` 四个系统模块化
- 将统计计算、实验状态和随机源从主仿真调度中拆分出来

### 阶段作用

这一阶段的作用是让系统具备“实验”而不是“试玩”的性质。  
实验条件可以重复，实验结果可以保存，后续也才能进行 A/B 比较、课题分析和结果复查。

### 可在报告中如何表述

项目在这一阶段完成了实验工作流建设，使仿真从演示型工具转变为可重复、可记录、可导出的实验平台。

---

## 六、阶段四：可视化与教学展示增强

### 阶段定位

这一阶段的任务是从“实验能跑”推进到“用户能看懂”，强化教学展示和结果解释能力。

### 阶段目标

- 让用户更容易理解三条基础规则与四大机制
- 让实验过程和 A/B 差异能被更直观地看到
- 让界面更适合课程展示和答辩说明

### 主要改动

- 新增 `Rules Lab`，独立演示分离、对齐、聚合三条基础规则
- 新增预设机制说明、关键观察指标和推荐 Seed 提示
- 新增双画布同屏对照，用于同 Seed 下的 A/B 现象展示
- 新增目标切换残影、捕获脉冲、状态着色等事件可视化
- 新增数据面板、事件时间线、批量波动对比图和自动 A/B 结论摘要
- 新增 README 中的结构图、机制 GIF 和规则 GIF，便于文档展示

### 阶段作用

这一阶段让系统不再只是给懂参数的人使用，而是具备了“教学解释界面”的性质。  
用户可以从规则演示、机制说明、过程图和同屏对照中直接理解系统想表达的内容。

### 可在报告中如何表述

为了提升系统的教学展示能力，项目在这一阶段增加了规则实验室、机制解释面板、时间线和同屏对照，使规则、过程和 A/B 差异都能被直观观察。

---

## 七、阶段五：A/B 一体化与无图形实验

### 阶段定位

这一阶段的任务是从“GUI 驱动实验”推进到“正式实验平台”，让实验单位和输出结构更适合课题研究。

### 阶段目标

- 把一次正式实验定义为一组 A/B 成对实验，而不是单条件运行
- 让批量实验走高速无渲染路径，只保留计算与输出
- 提供脱离 GUI 的实验 API 和脚本入口

### 主要改动

- 将正式实验单元改为一组 A/B 对照，共用同一个 Seed
- 将实验记录结构改为 `conditionA / conditionB / resultA / resultB / delta`
- 将批量实验改为无渲染高速计算，并输出 pair-level CSV
- 新增 `experiment-api.js` 与 `experiment-runner.js`，支持单次、批量和参数扫描实验
- 新增 `run-experiment.mjs` 脚本入口，支持不打开图形界面直接运行实验
- 新增 sweep 配置样例和 API 文档，支持单参数或多参数变化实验

### 阶段作用

这一阶段使项目从“带实验功能的展示系统”进一步升级为“可批量生产实验数据的实验平台”。  
后续做参数扫描、重复实验和课题数据整理时，不必依赖图形界面人工操作。

### 可在报告中如何表述

项目在这一阶段完成了 A/B 一体化实验重构，并建立了无图形实验 API 与脚本入口，使系统能够直接服务正式实验和数据生产。

## 八、当前状态总结

截至当前版本，系统已经从单画布演示原型演进为一个兼具以下能力的实验展示平台：

- 具有基础规则和抗捕食机制的可解释建模能力
- 具有固定 Seed、实验记录、批量实验和数据导出能力
- 具有规则教学、机制解释、同屏对照和时间线展示能力
- 具有 A/B 成对实验、无图形批量实验和参数扫描能力

当前较重要的核心指标包括：

| 指标 | 用途 |
|------|------|
| 实验捕获率 | 用于描述整个实验窗口内的整体风险水平 |
| 攻击成功率 | 用于描述捕食者锁定和截获效率 |
| 首次预警延迟 | 用于描述群体初始发现威胁的速度 |
| 警戒覆盖率 | 用于描述警戒是否扩散到全群 |
| 平均反应时间 | 用于描述群体响应效率 |
| 平均锁定时长 | 用于描述目标锁定稳定性 |
| 平均混淆强度 | 用于描述捕食者所受目标干扰程度 |
| 群体紧密度 | 用于描述群体是否聚拢 |
| 边缘被捕比例 | 用于描述边缘暴露风险 |

---

## 九、下一阶段可推进内容

如果继续开发，下一阶段可优先考虑以下方向：

- 增加更细的风险指标，例如单位个体风险、边缘/非边缘风险对比
- 增加更强的批量结果可视化，例如箱线图、误差棒图和热力图
- 增加实验结果本地持久化或项目内实验档案结构
- 增加更接近论文结构的实验流程页和结果展示页
- 增加更严格的局部安全中心模型或群体碎裂度分析

这些内容可以作为“下一阶段工作”单独写入报告中的展望部分。
