# 使用指南

## 环境要求

- Rust 工具链（rustc + cargo）
- wasm-pack
- 现代浏览器（Chrome / Firefox / Safari / Edge）
- 本地 HTTP 服务器（Python 3 自带 / npx serve 等）

## 构建步骤

### 1. 安装 Rust（如未安装）

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source "$HOME/.cargo/env"
```

### 2. 安装 wasm-pack（如未安装）

```bash
cargo install wasm-pack
```

### 3. 编译 WASM

```bash
cd boids-wasm
wasm-pack build --target web --release
```

编译产物在 `boids-wasm/pkg/` 目录下。

### 4. 启动本地服务器

回到项目根目录：

```bash
cd ..
python3 -m http.server 8080
```

### 5. 打开浏览器

访问 `http://localhost:8080/web/`

## 界面说明

### 画布区域（左侧）

- **蓝色三角形**: 普通鸟群个体
- **黄色三角形**: 处于警戒状态的鸟（检测到捕食者）
- **红色大三角形**: 捕食者
- **红色虚线**: 捕食者当前锁定目标的连线
- 左上角显示实时 FPS

### 控制面板（右侧）

#### 预设场景

四组 A/B 对比实验按钮，对应四大抗捕食机制。点击后自动调整所有参数并重启仿真。

#### 统计仪表盘

- **总捕获数**: 捕食者累计成功捕获次数
- **捕获率**: 近 5 秒平均每秒捕获次数
- **攻击成功率**: 成功捕获 / 攻击尝试次数
- **平均反应时间**: 从第一只鸟警觉到全群警觉的平均帧数
- **群体紧密度**: 所有鸟到群体质心的平均距离（越小越紧密）
- **边缘被捕比例**: 被捕鸟中，边缘个体占比
- **目标切换次数**: 捕食者累计切换追击目标的次数
- **存活鸟数**: 当前活跃的鸟群个体数

#### 参数滑块

可实时调整所有鸟群和捕食者参数，无需重启仿真。

#### 显示选项

- **显示感知范围**: 为每只鸟绘制视觉范围圆
- **显示尾迹**: 保留运动轨迹（半透明覆盖）
- **显示锁定线**: 显示捕食者到目标的连线
- **高亮边缘个体**: 距离群体边缘较远的个体变为黄色
- **捕食者攻击边缘**: 捕食者优先攻击距质心最远的个体

## 推荐实验流程

1. 打开页面，观察默认配置下的鸟群行为
2. 选择一个感兴趣的机制，点击 **A 预设**
3. 观察 30~60 秒，记录统计仪表盘数据
4. 切换到 **B 预设**，重复观察
5. 对比两组数据，验证理论预期
6. 尝试手动调整参数，探索极端情况

详细预设参数请参阅 [presets.md](presets.md)。
四大机制原理请参阅 [mechanisms.md](mechanisms.md)。
